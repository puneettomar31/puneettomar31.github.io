Deploying Laravel on Linux

Published Dec 2024 · by Puneet Tomar

Deployment is an essential part of web development. I have deployed multiple Laravel projects on Linux servers using FTP and SSH workflows during my time at Eglogics Softech and SciAstra.

Steps I Follow

  1. Set up Linux hosting environment (Apache/Nginx + PHP + MySQL)
  2. Upload Laravel files via FTP, Git pull, or rsync
  3. Configure .env with production database and mail settings
  4. Run composer install --no-dev --optimize-autoloader
  5. Run php artisan migrate --seed for DB setup
  6. Set correct folder permissions for storage and bootstrap/cache
  7. Configure virtual hosts or .htaccess rewrite rules for the public folder
  8. Run php artisan config:cache && php artisan route:cache for optimization

Nginx vs Apache

Nginx is generally faster for serving static assets and handles concurrent connections better. Apache with mod_rewrite works out of the box for Laravel's pretty URLs. For most shared hosting environments, Apache is more common.

Pro tip: Always test on a staging server before going live, and use version control for smooth, trackable deployments. Never make direct edits to production files.
